# One-Heliostat Train-Size Sensitivity Experiment

## Goal

Isolate a **single heliostat** from the 63-heliostat field and measure how the
number of training samples influences kinematic reconstruction quality.

The experiment runs one full perturb → train → evaluate cycle for each entry in
`TRAIN_SIZES = [1, 5, 10, 20, 25, 50, 75, 100]`.  Val and test sets are fixed
across all runs so the results are directly comparable.

A companion to `full_field_200_samples` (which fixes train=100 and scales to 63
heliostats).  Together they answer:
- How does recovery improve as more calibration images are available?
- Does the equifinality problem persist at low sample counts for a single heliostat?
- What is the minimum number of samples needed for reliable recovery?

## Heliostat

One heliostat is selected from the 63 in the full-field benchmark.

`HELIOSTAT_ID` in `config.py` (default `None` → auto-selects the first heliostat
found in the benchmark CSV that has local deflectometry data).  Override at
runtime with `--heliostat-id AA23`.

The chosen heliostat must be consistent between `create_scenario.py` and
`main.py` — both read `HELIOSTAT_ID` from `config.py`.

---

## Scenario

A new one-heliostat scenario is created at:
```
scenarios/one_heliostat_scenario/scenario.h5
```

Built by fitting NURBS surfaces to the heliostat's deflectometry data.
This takes ~30 seconds on GPU (vs ~10–30 min for 63 heliostats).

```
python create_scenario.py               # uses config.HELIOSTAT_ID (default: auto)
python create_scenario.py --heliostat-id AA23
python create_scenario.py --force       # overwrite existing
```

## Data

### Synthetic dataset

**No separate generation step.**  This experiment reuses the synthetic data
already generated by `full_field_200_samples`:

```
scenarios/full_field_200_samples_scenario/synthetic_data/
  {train|val|test}/
    {heliostat_id}/
      {idx:04d}/
        calibration_properties.json
        flux_image.png
```

Pre-requisite: `full_field_200_samples/generate_dataset.py` must have been run
and the chosen heliostat's folder must exist in each split.

| Split | Samples used | Source |
|-------|-------------|--------|
| train | 1–100 (swept) | first N entries from full-field synthetic train folder |
| val   | 50 (fixed)    | full-field synthetic val folder |
| test  | 50 (fixed)    | full-field synthetic test folder |

### Real PAINT dataset

Set `DATASET_TYPE = "real"` in `config.py`.  The PAINT calibration images for
the chosen heliostat are read from the benchmark split directory.  Kinematic
perturbations are skipped.

## Train-size sweep

`TRAIN_SIZES = [1, 5, 10, 20, 25, 50, 75, 100]` (configurable in `config.py`).

`main.py` runs one full train+eval cycle per size:
- Same perturbation (seed 42) applied each time → comparable baseline.
- Scenario reloaded from HDF5 for each size → clean state, no state leakage.
- Val and test mappings are identical across sizes.

---

## Evaluation

Same three-stage protocol as `full_field_200_samples`:

| Stage            | Scenario params | Expected result                  |
|------------------|-----------------|----------------------------------|
| Pre-perturbation | Clean           | ~0 mrad (sanity check)           |
| Post-perturbation| Perturbed       | High mrad                        |
| Post-training    | Trained         | Low mrad (recovery)              |

Two metrics reported: **focal-spot error (mrad)** and **pixel-wise L1**.

---

## Perturbations

Same parameter groups and ranges as `full_field_200_samples`:

| Parameter group                 | Range | Unit | Optimized? |
|---------------------------------|-------|------|------------|
| Rotation (4 joint tilts)        | ±5    | mrad | Yes        |
| Actuator initial angle aᵢ (×2)  | ±5    | mrad | Yes        |
| Actuator stroke length bᵢ (×2)  | ±5    | mm   | **No (frozen)** |
| Actuator offset cᵢ (×2)         | ±5    | mm   | Yes        |
| Translation deviation (9 joints)| ±50   | mm   | Yes        |
| Base position (E, N, U)         | ±50   | mm   | Yes        |

Perturbation seed: 42.  `n_heliostats=1` → single perturbation vector,
identical across all train-size runs.

## Hyperparameters

| Parameter               | Value  |
|-------------------------|--------|
| Learning rate (initial) | 1e-4   |
| Scheduler               | ReduceLROnPlateau (factor=0.5, patience=10) |
| Min LR                  | 1e-6   |
| Max epochs              | 100    |
| Batch size              | 8      |
| Early stopping patience | 400 (disabled — always runs fully) |
| Train rays              | 10     |
| Surface pts/facet       | 25 × 25 |

## Quick start

```bash
# 1. Create the one-heliostat scenario (one-time, ~30 s on GPU)
cd src/one_heliostat_train_sizes
python create_scenario.py

# 2. Run full sweep (~N × training time, roughly 5–30 min per size on GPU)
python main.py

# 3. Quick end-to-end smoke test (5 epochs, 2 sizes, ~2 min)
python main.py --smoke-test

# 4. Use a specific heliostat
python create_scenario.py --heliostat-id AA23
python main.py --heliostat-id AA23

# 5. Custom output directory
python main.py --output-dir outputs/my_custom_run
```
