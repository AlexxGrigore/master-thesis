# master-thesis

Kinematic reconstruction of heliostat fields using the [ARTIST](https://github.com/ARTIST-Association/ARTIST)
ray-tracing simulator and real PAINT calibration data.

## Experiments

### Experiment 1 — `one_heliostat_train_sizes`

**What it does:**
Kinematic reconstruction for a single heliostat, sweeping over multiple training data
sizes (default: `[1, 5, 10, 20, 25, 50, 75, 100]` samples). Each size runs a full
train + eval cycle under identical perturbations, making results directly comparable.
The goal is to understand how many calibration images are needed to recover kinematic
parameters to a given accuracy.

**Data:** reuses synthetic data pre-generated by `full_field_200_samples/generate_dataset.py`
— no separate data generation step is needed.

**Key config knobs** (`config.py`):
- `HELIOSTAT_ID` — fix to a specific heliostat or `None` to auto-select
- `TRAIN_SIZES` — list of training sample counts to sweep
- `DATASET_TYPE` — `"synthetic"` or `"real"`
- `LOSS_TYPE` — `"focal_spot"`, `"pixel"`, or `"alignment"`

**Run:**
```bash
cd src
python one_heliostat_train_sizes/main.py
python one_heliostat_train_sizes/main.py --heliostat-id AA23
python one_heliostat_train_sizes/main.py --smoke-test
```

### Experiment 2 — `full_training_pipeline`

**What it does:**
A coarse-to-fine pipeline for all 63 heliostats. A first stage applies the standard
kinematic reconstruction (coarse alignment), then a lightweight residual model
(linear, polynomial, or transformer) learns to predict the remaining per-sample
error from heliostat features and motor positions.

**Key config knobs** (`config.py`):
- `DATASET_TYPE` — `"synthetic"` or `"real"`
- `LOSS_TYPE` — `"focal_spot"`, `"pixel"`, or `"alignment"`
- `MAX_EPOCHS`, `LEARNING_RATE` — training hyperparameters

**Run:**
```bash
cd src
python full_training_pipeline/main.py
python full_training_pipeline/main.py --dataset-type real --model-type linear --daic
python full_training_pipeline/main.py --smoke-test
```

CLI flags: `--dataset-type`, `--model-type` (`linear` / `poly2` / `poly3` / `poly4` / `transformer`),
`--daic`, `--smoke-test`.

### Experiment 3 — `full_63_heli_kin_reconstruct` *(primary KR experiment)*

**What it does:**
Closed-loop kinematic reconstruction for all 63 heliostats with a corrected data
pipeline. The key difference from prior work:

> **Corrected pipeline:** perturbations are applied to the scenario *before* generating
> the synthetic dataset. The KR then starts from a clean (ideal) scenario and must
> learn the perturbation values from the data — the real inverse problem.

Previous experiments generated data from the clean scenario, applied perturbations,
then trained the KR to push parameters back to zero — a trivial task that did not
reflect reality.

**Two-stage training:**
1. **Stage 1 — AlignmentLoss** (no ray tracing, fast) — pre-trains joint angle parameters
   from motor-position measurements.
2. **Stage 2 — FocalSpotLoss or PixelLoss** (full ray tracing) — fine-tunes all kinematic
   parameters against the flux images.

**Outputs per run:**
- `convergence_stage1.png` / `convergence_stage2.png` — per-stage train/val loss
- `convergence.png` — combined convergence with accuracy reference lines
- `per_heliostat_accuracy_histogram.png` — distribution of post-training FSE across heliostats
- `per_heliostat_accuracy.png` / `.json` — per-heliostat accuracy table
- `flux_grid_best_{hid}.png` / `flux_grid_worst_{hid}.png` — 10×5 grid of measured vs
  predicted flux images for the best and worst heliostats on the test set
- `field_accuracy_map.png` — ENU field scatter plot, heliostats coloured green/yellow/red
  by accuracy (thresholds: 1.5 mrad, 2.5 mrad)
- `summary_table.png` — val and test mean + median accuracy
- `kinematic_parameters.json`, `results.json`, `timing.json` — full numerical results
- `recovery_*.png` — parameter recovery plots (synthetic runs only)

**Data generation** (run once, or after changing perturbation ranges):
```bash
cd src
python full_63_heli_kin_reconstruct/generate_dataset.py
python full_63_heli_kin_reconstruct/generate_dataset.py --force   # overwrite existing
python full_63_heli_kin_reconstruct/generate_dataset.py --daic    # DAIC paths
```

**Run locally:**
```bash
cd src
python full_63_heli_kin_reconstruct/main.py --dataset-type synthetic --loss-type focal_spot \
    --stage1-epochs 100 --stage2-epochs 300
python full_63_heli_kin_reconstruct/main.py --smoke-test
```

**Run on DAIC** — use the sbatch files (see below).

## `src/artist_extensions`

Custom extensions to ARTIST that must not be edited inside the ARTIST package itself.
All overrides live here so ARTIST stays a clean dependency.

### `kinematic_reconstructors.py`

Subclasses of ARTIST's `KinematicsReconstructor` implementing the parameter setup
from Wortberg (2025), Table 5.3:

| Class | Stage | Loss |
|---|---|---|
| `WortbergKinematicReconstructor` | Stage 2 | `FocalSpotLoss` |
| `WortbergPixelReconstructor`     | Stage 2 | `PixelLoss` / `BlurredPixelLoss` |
| `WortbergAlignmentReconstructor` | Stage 1 | `AlignmentLoss` |

All three variants optimise the same parameter set:
- **Translation deviations** — 9 joint + concentrator translations (±50 mm)
- **Rotation deviations** — 4 joint tilts (±5 mrad)
- **Actuator initial angle** `a_i` — (±5 mrad)
- **Actuator offset** `c_i` — (±5 mm)
- **Base position deviation** `(e, n, u)` — (±50 mm), injected into
  `active_heliostat_positions` each epoch since ARTIST does not handle this parameter
  natively

**Frozen:** actuator stroke length `b_i`.

### `loss_functions_ext.py`

- `AlignmentLoss` — MSE on motor positions converted to joint angles; no ray tracing,
  used for Stage 1 pre-training
- `BlurredPixelLoss` — Gaussian blur (σ=1) → peak-normalise → MSE; smoother gradients
  than raw pixel MSE
- `PixelLossL1` — L1 variant of ARTIST's `PixelLoss`; less sensitive to bright specular
  outlier pixels

### `cached_paint_parser.py`

`CachedPaintCalibrationDataParser` — drop-in replacement for ARTIST's
`PaintCalibrationDataParser` that caches parsed tensors in CPU RAM after the first
epoch. On DAIC, calibration data lives on NFS (`/tudelft.net`); re-reading thousands
of files over the network every epoch causes significant slowdown. This parser
eliminates that overhead from epoch 2 onward.
